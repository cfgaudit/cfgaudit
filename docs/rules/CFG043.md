# CFG043 — `permissions.deny` does not restrict cloud provider credential files

**Severity:** `error`
**OWASP:** [LLM02:2025 – Sensitive Information Disclosure](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM02_2025-Sensitive_Information_Disclosure.html)

## What cfgaudit checks

Cloud provider credential files grant programmatic access to infrastructure. cfgaudit flags a `permissions.deny` block that **exists** but does not cover the well-known credential paths for one or more providers:

| Provider | Paths |
|----------|-------|
| AWS | `**/.aws/credentials`, `**/.aws/config` |
| GCP | `**/.config/gcloud/`, `**/application_default_credentials.json` |
| Azure | `**/.azure/`, `**/.azure/accessTokens.json` |

At least one pattern per provider must be present; a broad pattern like `Read(~/.aws/*)` counts as covering AWS. The finding lists exactly which providers are uncovered. A completely absent `permissions.deny` block is reported by CFG006.

## Trigger example

```json
{
  "permissions": {
    "deny": ["Read(.env)", "Read(**/.aws/credentials)"]
  }
}
```

→ `error` (GCP and Azure still uncovered).

## Safe alternative

```json
{
  "permissions": {
    "deny": [
      "Read(**/.aws/credentials)",
      "Read(**/.aws/config)",
      "Read(**/.config/gcloud/**)",
      "Read(**/application_default_credentials.json)",
      "Read(**/.azure/**)"
    ]
  }
}
```
