# CFG062 — Gemini CLI allows unrestricted Git extensions (`security.blockGitExtensions`)

**Severity:** `warn`
**OWASP:** [LLM03:2025 – Supply Chain](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM03_2025-Supply_Chain.html)

## What cfgaudit checks

Gemini CLI can install extensions from Git repositories. `security.blockGitExtensions: true` blocks that; `security.allowedExtensions` constrains which extensions are permitted. A committed `.gemini/settings.json` that **explicitly** sets `blockGitExtensions: false` **and** provides no `allowedExtensions` allow-list lets the workspace install extensions from arbitrary Git repos — a supply-chain vector, since extension code runs with the agent's privileges.

cfgaudit flags this only when `blockGitExtensions` is **explicitly `false`** (not merely absent) and `allowedExtensions` is empty, in `.gemini/settings.json` (project) and `~/.gemini/settings.json` (with `--user`).

## Trigger example

```json
{ "security": { "blockGitExtensions": false } }
```

→ `warn` (no `allowedExtensions` to constrain installs).

## Safe alternative

Block Git extensions, or pin an allow-list:

```json
{ "security": { "blockGitExtensions": true } }
```
```json
{ "security": { "blockGitExtensions": false, "allowedExtensions": ["github.com/your-org/.*"] } }
```
