# CFG012 — `settings.json` structure does not match the official schema

**Severity:** `warn`
**OWASP:** [LLM02:2025 – Sensitive Information Disclosure](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM02_2025-Sensitive_Information_Disclosure.html)

## What cfgaudit checks

cfgaudit ships with an embedded snapshot of the [SchemaStore Claude Code settings schema](https://json.schemastore.org/claude-code-settings.json) and validates the top level of `settings.json` against it. Two sub-checks fire as `warn`-level findings:

1. **Unknown top-level keys** — any key that is not defined in the bundled schema. Catches typos, stale config from old Claude Code versions, and injected keys an attacker added hoping nobody would notice.
2. **Type mismatch** — values whose JSON type does not match the schema (e.g. `apiKeyHelper` set to an array instead of a string). Type mismatches often indicate malformed values or an attempt to confuse downstream parsers.

### Schema lag

The published schema lags behind CLI releases. A key absent from the schema is not necessarily invalid — it may have been added upstream in a recent Claude Code version. For that reason, CFG012 is always `warn`, never `error`. If you know the warning is a false positive, suppress it with a `// cfgaudit:ignore CFG012` comment.

A small allowlist accommodates keys that are widespread in real configurations but absent from the schema today (currently `mcpServers` and `defaultMode`). Update it in `internal/schema/schema.go` as the upstream schema catches up.

### Strict-parse keys

For keys the internal Go parser models (`permissions`, `env`, `hooks`, `mcpServers`), a type mismatch is caught by the JSON unmarshaller before any rule runs — cfgaudit exits with code 2 and a parser error in that case. CFG012 catches type mismatches for the remaining keys (everything that flows through the raw map).

## Trigger examples

Unknown key:

```json
{
  "thisKeyDoesNotExist": true
}
```

Type mismatch:

```json
{
  "apiKeyHelper": ["should be a string, not an array"]
}
```

## Safe alternative

Keep `settings.json` to keys documented in the [Claude Code settings reference](https://docs.anthropic.com/en/docs/claude-code/settings) and use the JSON types the schema specifies. If you intentionally use a newer key not yet in the bundled schema, suppress the warning inline.
