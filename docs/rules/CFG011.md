# CFG011 — MCP server `alwaysAllow` is too broad

**Severity:** `warn`
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

cfgaudit inspects each `mcpServers.*.alwaysAllow` list in `settings.json` and flags when it is too permissive. `alwaysAllow` skips the per-call confirmation prompt — that is only appropriate for read-only, low-risk tools.

A finding is emitted when the list:

1. **Contains the wildcard `"*"`** — auto-approves every tool the server exposes, including ones added in future updates.
2. **Includes a state-mutating tool name** (substring match, case-insensitive): `write`, `delete`, `remove`, `edit`, `exec`, `run_command`, `shell`, `bash`, `kill`, `patch`, `move_file`, `rename`, `create_file`, `create_directory`.
3. **Has 10 or more entries** — even when each entry is benign, blanket auto-approval at this scale is rarely intentional.

The first matching condition reports; once the wildcard fires there is no point listing individual tools.

## Trigger example

```json
{
  "mcpServers": {
    "shell": {
      "command": "mcp-server-shell",
      "alwaysAllow": ["*"]
    },
    "fs": {
      "command": "mcp-server-filesystem",
      "alwaysAllow": ["read_file", "write_file", "delete_file"]
    }
  }
}
```

## Safe alternative

Limit `alwaysAllow` to truly read-only tools and let the user confirm everything else:

```json
{
  "mcpServers": {
    "fs": {
      "command": "mcp-server-filesystem",
      "alwaysAllow": ["read_file", "list_directory"]
    }
  }
}
```
