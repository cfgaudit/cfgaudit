# CFG040 — `permissions.allow` contains unrestricted WebFetch

**Severity:** `warn`
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

cfgaudit flags a `permissions.allow` entry that grants **unrestricted** WebFetch — Claude can then fetch any URL, an exfiltration channel for data or secrets in the environment. Flagged forms:

- `WebFetch` (bare)
- `WebFetch()` / `WebFetch(*)`
- `WebFetch(domain:*)` / `WebFetch(url:*)`

Scoped entries like `WebFetch(domain:api.example.com)` are **not** flagged.

> Note: restricting WebFetch alone does not stop network access if `Bash(curl *)` / `Bash(wget *)` is also allowed — those are covered by **CFG023**.

## Trigger example

```json
{ "permissions": { "allow": ["WebFetch(domain:*)"] } }
```

## Safe alternative

```json
{ "permissions": { "allow": ["WebFetch(domain:api.example.com)"] } }
```
