# CFG050 ‚Äî hardcoded secret in an MCP server env or headers

**Severity:** `error`
**OWASP:** [LLM02:2025 ‚Äì Sensitive Information Disclosure](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM02_2025-Sensitive_Information_Disclosure.html)
**OWASP MCP:** [MCP01:2025 ‚Äì Token Mismanagement & Secret Exposure](https://owasp.org/www-project-mcp-top-10/) ‚Äî provisional (MCP Top 10 v0.1)

## What cfgaudit checks

CFG007 detects hardcoded secrets in the **settings.json `env`** block. An MCP server carries its own credentials in its **`env`** map and, for remote (HTTP/SSE/WebSocket) transports, in its request **`headers`** (e.g. `Authorization: Bearer ‚Ä¶`). A committed `.mcp.json` / `mcpServers` entry with a baked-in token is a direct credential leak ‚Äî to the repo, and (for headers) to the remote endpoint on every request.

cfgaudit applies the CFG007 secret heuristics across every MCP source it discovers (settings.json, `.mcp.json`, other agents' configs):

- **`env`** values matching a vendor key pattern (Anthropic, OpenAI, GitHub, GitLab, AWS, Slack, Google), or keys with a secret-like name (`*_TOKEN`, `*_SECRET`, `*_API_KEY`, ‚Ä¶) holding a literal value.
- **`headers`** values matching a vendor pattern, and auth headers (`Authorization`, `Proxy-Authorization`, `X-Api-Key`, `X-Auth-Token`, ‚Ä¶) carrying a literal credential (after stripping a `Bearer`/`Basic`/`Token` scheme word).

The same header check runs over the **`headers` of a Copilot `type: "http"` hook** (`.github/hooks/*.json`), which is a committed request-credential block of exactly the same shape. `disableAllHooks: true` does *not* suppress it ‚Äî a secret committed into a file has leaked whether or not the agent currently runs the hook. The outbound channel itself is [CFG088](CFG088.md).

A reference to an environment variable is the *correct* pattern and is **not** flagged: `${TOKEN}` / `$TOKEN`, plus obvious placeholders (`<your-token>`, `changeme`, `xxxxx`, `your-‚Ä¶`, `TODO`, `example`).

## Trigger example

```json
{
  "mcpServers": {
    "remote": {
      "url": "https://mcp.example/sse",
      "headers": { "Authorization": "Bearer sk-ant-abcdef1234567890" },
      "env": { "GITHUB_TOKEN": "ghp_abcdefghij0123456789ABCDEF" }
    }
  }
}
```

‚Üí two `error` findings. Using `"Authorization": "Bearer ${MCP_TOKEN}"` and `"GITHUB_TOKEN": "${GITHUB_TOKEN}"` clears both.

## Safe alternative

Reference an environment variable so the secret stays out of the committed config:

```json
{ "mcpServers": { "remote": { "url": "https://mcp.example/sse", "headers": { "Authorization": "Bearer ${MCP_TOKEN}" } } } }
```
