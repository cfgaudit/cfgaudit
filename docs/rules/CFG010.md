# CFG010 — MCP server uses an unpinned package or image version

**Severity:** `warn`
**OWASP:** [LLM03:2025 – Supply Chain Vulnerabilities](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM03_2025-Supply_Chain.html)

## What cfgaudit checks

cfgaudit inspects each MCP server entry — both the inline `mcpServers` block in `settings.json` **and** the project's `.mcp.json` (the file `enableAllProjectMcpServers` / `enabledMcpjsonServers` auto-approve) — and flags any of:

1. **An arg ending in `@latest` or `:latest`** — covers both npm-style package pins (`@scope/pkg@latest`) and container image tags (`registry/image:latest`).
2. **An npm-style runner (`npx`, `pnpm dlx`, `yarn dlx`, `bunx`) whose first positional package argument has no `@<version>` suffix** — e.g. `npx -y @modelcontextprotocol/server-filesystem`. The package resolves to whatever the registry currently calls `latest`.

Local-path arguments (`./pkg`, `../pkg`, `/pkg`) are not flagged.

An upstream compromise of an unpinned dependency silently propagates to every developer who reloads the agent — there is no SCA tooling between the registry and the MCP server.

## Trigger example

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem@latest", "/tmp"]
    }
  }
}
```

## Safe alternative

Pin to an exact version:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem@1.2.3", "/tmp"]
    }
  }
}
```

For container images, pin to an immutable digest where possible:

```
ghcr.io/owner/server@sha256:abcd…
```
