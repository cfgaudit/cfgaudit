# CFG065 — Continue config has a hardcoded inline `apiKey`

**Severity:** `error`
**OWASP:** [LLM02:2025 – Sensitive Information Disclosure](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM02_2025-Sensitive_Information_Disclosure.html)

## What cfgaudit checks

[Continue](https://github.com/continuedev/continue)'s `config.yaml` configures model providers and MCP servers, each of which can carry an inline `apiKey`. A committed `.continue/config.yaml` with a real key literal leaks that credential to anyone with repo access.

cfgaudit flags a hardcoded `apiKey` literal on:

- a `models[]` entry (provider credential), and
- a remote `mcpServers[]` entry (sse / streamable-http server credential)

in `.continue/config.yaml` (project) and `~/.continue/config.yaml` (with `--user`).

### Not flagged

Continue's reference syntax is the safe pattern and does **not** fire: `${{ secrets.NAME }}` / `${{ inputs.NAME }}` templates, an `$ENV`-style reference, the `continue-proxy` provider's `apiKeyLocation`, and obvious placeholders (`YOUR_API_KEY`, `<...>`, `changeme`, …). Values under 8 characters are ignored.

## Trigger example

```yaml
models:
  - name: gpt
    provider: openai
    apiKey: sk-proj-AbCdEf0123456789
```

→ `error`.

## Safe alternative

```yaml
models:
  - name: gpt
    provider: openai
    apiKey: ${{ secrets.OPENAI_API_KEY }}
```
