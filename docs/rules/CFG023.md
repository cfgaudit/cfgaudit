# CFG023 — `permissions.allow` grants a dangerous command with open-ended args

**Severity:** `error` (network / privilege / remote-pkg / shell) · `warn` (interpreters, exec-via-flags, lateral movement)
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

cfgaudit inspects every `permissions.allow` entry of the form `Bash(<command> …*…)` — i.e. one that grants the command with **open-ended (wildcard) arguments** — and flags binaries that hand far more capability than the entry appears to. CFG001 already covers the catch-all `Bash(*)`; this rule covers the specific commands below.

A finding fires **only when the entry contains a wildcard**. An exactly-pinned command such as `Bash(git status)` or `Bash(npm test)` is exempt — the user spelled out precisely what may run.

| Category | Severity | Binaries | Why |
|----------|----------|----------|-----|
| Network egress | `error` | `curl`, `wget` | exfiltrate data / fetch and run remote payloads |
| Privilege escalation | `error` | `sudo`, `doas` | run as root |
| Remote package exec | `error` | `npx`, `bunx` | run arbitrary packages from a registry |
| Shell interpreter | `error` | `bash`, `sh`, `dash`, `zsh`, `ksh`, `csh`, `tcsh`, `fish` | open-ended args = arbitrary command execution |
| Language interpreter | `warn` | `python`, `python3`, `perl`, `ruby`, `node`, `deno` | open-ended args can execute arbitrary code |
| Exec-via-flags | `warn` | `find`, `sed`, `awk`, `gawk`, `xargs`, `env`, `tar`, `git` | `find -exec`, GNU `sed e///`, `awk system()`, `env`/`xargs <cmd>`, `tar --checkpoint-action`, `git -c core.pager=…` all run arbitrary commands |
| Lateral movement | `warn` | `ssh`, `scp`, `rsync` | remote command execution / data movement to other hosts |

Standard build/dev tooling (`npm`, `pip`, `docker`, `make`, `go`, `cargo`, `yarn`, …) is **not** flagged: allow-listing those with arguments is normal, deliberate practice.

## Trigger example

```json
{
  "permissions": {
    "allow": ["Bash(curl *)", "Bash(find *)"]
  }
}
```

`Bash(curl *)` → `error` (unrestricted network). `Bash(find *)` → `warn` (`find -exec` runs anything).

## Safe alternative

Pin the exact command and arguments you actually need, with no wildcard:

```json
{ "permissions": { "allow": ["Bash(git status)", "Bash(curl https://api.internal.example/health)"] } }
```
