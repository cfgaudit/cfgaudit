# CFG003 — `enableAllProjectMcpServers: true`

**Severity:** `error`  
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)  
**OWASP MCP:** [MCP02:2025 – Privilege Escalation via Scope Creep](https://owasp.org/www-project-mcp-top-10/) — provisional (MCP Top 10 v0.1)
**CVE:** CVE-2025-59536 (CVSS 8.7)

## What cfgaudit checks

cfgaudit flags `enableAllProjectMcpServers: true` in any `settings.json`. This setting auto-approves every MCP server defined in any `.mcp.json` discovered in the repository — including ones added by anyone with commit access. A malicious `.mcp.json` committed to a shared repository executes automatically when another developer opens the project, without any confirmation prompt.

CVE-2025-59536 was exploited via exactly this vector.

## Trigger example

```json
{
  "enableAllProjectMcpServers": true
}
```

## Safe alternative

Use an explicit allowlist of trusted server names instead:

```json
{
  "enabledMcpjsonServers": ["github", "memory"]
}
```
