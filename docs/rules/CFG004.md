# CFG004 — `defaultMode` set to `bypassPermissions` or `auto`

**Severity:** `error` (bypassPermissions) / `warn` (auto)  
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

cfgaudit flags `permissions.defaultMode` when set to `"bypassPermissions"` or `"auto"`. This is the schema location for the key — a top-level `defaultMode` is not honoured by Claude Code and is not flagged.

| Value | Effect | Severity |
|-------|--------|----------|
| `"bypassPermissions"` | All permission checks disabled; Claude Code acts with full autonomy and no confirmation prompts. All subagents inherit this mode with no override. | `error` |
| `"auto"` | Confirmation prompts suppressed; permission rules still enforced, but any allow rule is acted on immediately. | `warn` |

Real-world incidents with `bypassPermissions` include developers losing entire home directories when Claude Code executed `rm -rf ~/` without interruption.

## Trigger examples

```json
{ "permissions": { "defaultMode": "bypassPermissions" } }
```

```json
{ "permissions": { "defaultMode": "auto" } }
```

## Safe alternative

Omit `defaultMode` entirely (interactive mode is the default), or set it explicitly:

```json
{ "permissions": { "defaultMode": "default" } }
```

Enable `bypassPermissions` only in isolated, throwaway environments (containers, ephemeral VMs) where destructive actions have no lasting impact.
