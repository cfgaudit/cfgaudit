# CFG071 — model/provider base URL sends the API key over cleartext

**Severity:** `error`
**OWASP:** [LLM02:2025 – Sensitive Information Disclosure](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM02_2025-Sensitive_Information_Disclosure.html)

## What cfgaudit checks

Multi-provider AI tools let you point a model at a custom endpoint via a base URL — the provider API key is sent there. cfgaudit flags such a base URL when it uses **cleartext (`http://` / `ws://`) to a non-loopback host**, because the key then travels unencrypted to a remote endpoint:

- **Continue** — `models[].apiBase` in `.continue/config.yaml`
- **Codex** — `chatgpt_base_url` and `[model_providers.<name>].base_url` in `~/.codex/config.toml`

This is the multi-provider analogue of [CFG005](CFG005.md) (`ANTHROPIC_BASE_URL`).

### Why only cleartext

Unlike Claude Code (whose only valid endpoint is `api.anthropic.com`), Continue and Codex are **multi-provider by design** — Azure OpenAI, OpenRouter, internal gateways, and local Ollama are all legitimate custom endpoints. So a non-default endpoint is **not** flagged on its own; only the unambiguous cleartext-to-remote case (key sent in plaintext) is. `https://`/`wss://` endpoints and loopback addresses are not flagged.

## Trigger example

```yaml
models:
  - name: gpt
    provider: openai
    apiBase: http://gateway.internal.example/v1
    apiKey: ${{ secrets.OPENAI_API_KEY }}
```

→ `error` (the key is sent over cleartext `http://` to a remote host).

## Safe alternative

```yaml
models:
  - name: gpt
    provider: openai
    apiBase: https://gateway.internal.example/v1
```

(A loopback endpoint such as `http://localhost:11434/v1` for a local Ollama is also fine.)
