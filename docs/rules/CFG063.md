# CFG063 — Codex CLI auto-approves commands (`approval_policy`)

**Severity:** `error` (`never`) · `warn` (`on-failure`)
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

OpenAI Codex CLI's `approval_policy` (in `~/.codex/config.toml`) controls when the agent asks before running commands — the analog of Claude Code's `defaultMode`. Two values remove the human-in-the-loop:

| Value | Meaning | Severity |
|-------|---------|----------|
| `never` | Never ask; **all** commands are auto-approved (non-interactive) — the Codex equivalent of `defaultMode: bypassPermissions` | `error` |
| `on-failure` | DEPRECATED — all commands are auto-approved and only escalated to the user **on failure** | `warn` |

The safe values are `untrusted` (only known-safe read-only commands auto-approved) and `on-request`. cfgaudit checks `~/.codex/config.toml` (with `--user`).

## Trigger example

```toml
approval_policy = "never"
```

→ `error`.

## Safe alternative

```toml
approval_policy = "on-request"
```

Keep approval interactive; reserve `never` for sandboxed, non-interactive CI runs where you also constrain `sandbox_mode` (see CFG064).
