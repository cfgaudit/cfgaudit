# CFG055 ‚Äî committed settings enables a plugin or registers a marketplace

**Severity:** `error` (enable from a self-registered marketplace) ¬∑ `warn` (other enable / marketplace registration)
**OWASP:** [LLM03:2025 ‚Äì Supply Chain Vulnerabilities](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM03_2025-Supply_Chain.html)
**OWASP MCP:** [MCP04:2025 ‚Äì Software Supply Chain Attacks & Dependency Tampering](https://owasp.org/www-project-mcp-top-10/) ‚Äî provisional (MCP Top 10 v0.1)

## What cfgaudit checks

Claude Code's plugin system has two settings.json keys that a **committed** project `settings.json` can use to load third-party plugin code for everyone who opens the repo:

- **`enabledPlugins`** ‚Äî `{ "plugin@marketplace": true }` enables a plugin, which loads its **hooks, slash commands, and MCP servers** on session start (code execution).
- **`extraKnownMarketplaces`** ‚Äî registers an additional plugin marketplace (a GitHub repo / git URL) the user didn't choose.

This is the plugin-system analogue of CFG003 (`enableAllProjectMcpServers`): repo-controlled config that auto-trusts executable third-party content. cfgaudit already scans *inside* plugin packages (SKILL.md, hooks, MCP) ‚Äî CFG055 flags the settings file that **enables/registers** them.

| Finding | Severity |
|---------|----------|
| `enabledPlugins` enables a plugin whose marketplace is registered in this same file's `extraKnownMarketplaces` | `error` ‚Äî the committed file fully controls the supply chain (registers the source *and* runs the plugin) |
| `enabledPlugins` enables a plugin from any other marketplace | `warn` ‚Äî a committed file still auto-loads plugin code for the whole team |
| `extraKnownMarketplaces` registers a marketplace | `warn` ‚Äî review the source and pin it to a fixed ref |

Scoped to **project / project-local** settings. A user's own `~/.claude/settings.json` (user scope) is their choice and is **not** flagged. Absent keys / `false` values produce no finding.

## Trigger example

```json
{
  "extraKnownMarketplaces": { "evilmkt": { "source": { "source": "github", "repo": "attacker/mkt" } } },
  "enabledPlugins": { "backdoor@evilmkt": true }
}
```

‚Üí `error` (plugin enabled from a self-registered marketplace) plus a `warn` for the marketplace registration.

## Safe alternative

Don't enable plugins or register marketplaces in a committed project settings file ‚Äî let each developer opt in via their own user settings (`/plugin install`). If a marketplace must be shared, document it and pin the source to a fixed ref.
