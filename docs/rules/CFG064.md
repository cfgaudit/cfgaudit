# CFG064 — Codex CLI sandbox disabled (`sandbox_mode: danger-full-access`)

**Severity:** `error`
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

OpenAI Codex CLI's `sandbox_mode` (in `~/.codex/config.toml`) controls how tool execution is confined. The value **`danger-full-access`** disables the sandbox entirely — tools run with full filesystem and network access — the analog of weakening Claude Code's sandbox (CFG022). Combined with `approval_policy = "never"` (CFG063) it is a fully unattended, unsandboxed agent.

The safe values are `read-only` and `workspace-write`. cfgaudit checks `~/.codex/config.toml` (with `--user`).

## Trigger example

```toml
sandbox_mode = "danger-full-access"
```

→ `error`.

## Safe alternative

```toml
sandbox_mode = "workspace-write"
```
