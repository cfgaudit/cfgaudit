# CFG002 — `permissions.allow` contains unrestricted file-write pattern

**Severity:** `warn`  
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

cfgaudit flags `permissions.allow` entries that match `Edit(*)` or `Write(*)` — patterns that let the agent overwrite any file in the workspace without a confirmation prompt. Combined with a prompt injection attack, this allows an adversary to corrupt source files, configuration, or secrets.

## Trigger example

```json
{
  "permissions": {
    "allow": ["Edit(*)", "Write(*)"]
  }
}
```

## Safe alternative

Scope to the directories the agent actually needs to modify:

```json
{
  "permissions": {
    "allow": ["Edit(src/**)", "Write(output/*)"]
  }
}
```
