# CFG051 — skill/command/subagent grants overly broad `allowed-tools`

**Severity:** `error` (all-tools `*`/`all`) · `warn` (unrestricted shell `Bash`)
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

Claude Code custom **slash commands** (`.claude/commands/*.md`), **subagents** (`.claude/agents/*.md`), and plugin **skills** (`SKILL.md`) declare which tools they may use in a YAML frontmatter `allowed-tools` field. A committed command/skill that grants unrestricted shell or all tools can run anything the moment it is invoked — the frontmatter analogue of the permission-breadth rules (CFG002/CFG006).

cfgaudit parses the frontmatter and flags an `allowed-tools` grant that is too broad:

| Grant | Result |
|-------|--------|
| `*`, `all`, `full` | `error` — unrestricted tool access |
| `Bash` (bare, no `Bash(cmd)` restriction), `Bash(*)`, `shell`, `execute` | `warn` — unrestricted shell access (broad but commonly legitimate) |

A grant that is cancelled by a sibling `disallowed-tools` entry is not flagged. A scoped grant such as `Bash(npm test)` or a specific tool list (`Read`, `Edit`) is fine. Files without `allowed-tools` frontmatter (plain CLAUDE.md, `.cursorrules`, …) produce no finding.

## Trigger example

```markdown
---
description: Deploy the app
allowed-tools: Bash, Read
---
Run the deploy script.
```

→ `warn` (bare `Bash` = unrestricted shell; broad but often legitimate). Narrow it when you can:

```markdown
---
description: Deploy the app
allowed-tools: Bash(./deploy.sh), Read
---
```

## Safe alternative

Grant only the specific tools (and specific Bash commands) the command needs, or use `disallowed-tools` to remove dangerous ones from a broader set.
