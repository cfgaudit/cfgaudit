## What cfgaudit checks

cfgaudit scans every command-bearing key in `settings.json` â€” hooks (`hooks.*[].hooks[].command`) **and** the credential/runtime helpers Claude Code also executes (`apiKeyHelper`, `awsCredentialExport`, `awsAuthRefresh`, `gcpAuthRefresh`, `otelHeadersHelper`, `statusLine.command`, `fileSuggestion.command`) **and** each MCP server's `headersHelper` â€” and flags any reference to a shell variable: `$NAME` / `${NAME}` (Bourne shells) or `%NAME%` (Windows cmd.exe).

Hook commands run with the privileges of the Claude Code process and fire automatically on lifecycle events. Whenever part of a hook command is interpolated from a variable, that value may originate from agent output, tool output, or filenames the agent chose â€” i.e. attacker-influenced content. Crafted input can break out of unquoted arguments or escape quotes when shells re-parse strings, turning the hook into an arbitrary-command-execution sink.

Command substitution (`$(...)`) and positional parameters (`$1`, `$@`, `$?`) are intentionally **not** matched by this rule â€” they are tracked separately (issue #38).

The finding is `warn` for project and project-local scope, but escalates to `error` for a user-global `~/.claude/settings.json` â€” a malicious hook there fires on every project the user opens.

## Trigger example

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "*",
        "hooks": [
          { "type": "command", "command": "notify-send \"$TOOL_OUTPUT\"" }
        ]
      }
    ]
  }
}
```

## Safe alternative

Prefer fixed commands. When the hook genuinely needs context, pass it via stdin rather than as a shell-expanded argument:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "*",
        "hooks": [
          { "type": "command", "command": "/usr/local/bin/notify-from-stdin" }
        ]
      }
    ]
  }
}
```

If you must interpolate, ensure the value is single-quoted on the shell side and the receiving program treats the argument as untrusted data, not as a shell fragment.
