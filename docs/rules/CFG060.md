# CFG060 — Gemini CLI auto-approves tool actions (`defaultApprovalMode`)

**Severity:** `error`
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

Gemini CLI's approval mode (`general.defaultApprovalMode` in `settings.json`) is the analog of Claude Code's `defaultMode`. The value **`auto_edit`** auto-approves every edit tool without prompting (and `yolo`, should it appear as a mode, approves everything). A project `.gemini/settings.json` committing `auto_edit` is the Gemini equivalent of `defaultMode: bypassPermissions` (CFG004): it removes the human-in-the-loop for everyone who opens the workspace.

cfgaudit flags `general.defaultApprovalMode` set to `auto_edit` (or `yolo`), in `.gemini/settings.json` (project) and `~/.gemini/settings.json` (with `--user`).

## Trigger example

```json
{ "general": { "defaultApprovalMode": "auto_edit" } }
```

→ `error`. The safe values are `default` (prompt for approval) and `plan` (read-only).

## Safe alternative

```json
{ "general": { "defaultApprovalMode": "default" } }
```

Leave approval interactive; do not commit an auto-approving mode into a shared workspace config.
