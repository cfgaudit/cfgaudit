# CFG044 — `permissions.deny` does not restrict SSH private keys

**Severity:** `error`
**OWASP:** [LLM02:2025 – Sensitive Information Disclosure](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM02_2025-Sensitive_Information_Disclosure.html)

## What cfgaudit checks

SSH private keys grant access to remote systems and are a prime exfiltration target. cfgaudit flags a `permissions.deny` block that **exists** but does not cover them. Coverage is satisfied by either:

- a `.ssh` directory pattern — `Read(**/.ssh/**)`, `Read(~/.ssh/*)`, `.ssh/*`, or
- a well-known private-key filename — `id_rsa`, `id_ed25519`, `id_ecdsa`, `id_dsa`.

A completely absent `permissions.deny` block is reported by CFG006, not here.

## Trigger example

```json
{
  "permissions": {
    "deny": ["Read(.env)", "Read(**/*.pem)"]
  }
}
```

→ `error` (no `.ssh` / `id_*` coverage).

## Safe alternative

```json
{ "permissions": { "deny": ["Read(**/.ssh/**)"] } }
```

or explicitly:

```json
{
  "permissions": {
    "deny": ["Read(**/id_rsa)", "Read(**/id_ed25519)", "Read(**/id_ecdsa)", "Read(**/id_dsa)"]
  }
}
```
