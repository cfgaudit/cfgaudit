# CFG045 — ShellCheck analysis of hook / helper commands

**Severity:** `error` / `warn` / `info` (mapped from ShellCheck)
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)

## What cfgaudit checks

The regex-based command rules (CFG008/009/014/015/027/028/037/038/039) catch *structural* patterns, but miss a large class of shell-level issues that need real grammar awareness — unquoted expansions, unsafe `eval`, dangerous pipes, etc. CFG045 runs [ShellCheck](https://www.shellcheck.net/) over each command site (hooks **and** the command-running helpers) and surfaces its diagnostics, with the ShellCheck code (`SC2086`, `SC2046`, …) in the message.

ShellCheck severities map to cfgaudit severities: `error → error`, `warning → warn`, `info`/`style → info`.

## Opt-in & requirements

CFG045 is **off by default**. Enable it with the `--shellcheck` flag or `shellcheck: true` in `.cfgaudit.yml`. It requires the `shellcheck` binary on `PATH`; when ShellCheck is requested but not installed, cfgaudit logs a warning to stderr and continues without shell analysis (no finding).

```sh
cfgaudit --shellcheck
```

```yaml
# .cfgaudit.yml
shellcheck: true
```

## Example

A hook command `rm $FILE` yields an `info` finding referencing `SC2086` ("Double quote to prevent globbing and word splitting").

## Relationship to other rules

Complements, not replaces, the regex command rules — those run unconditionally and independently of ShellCheck availability; CFG045 adds deeper, grammar-aware analysis when enabled.
