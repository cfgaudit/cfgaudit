# CFG053 — blanket MCP-trust settings beyond `enableAllProjectMcpServers`

**Severity:** `error` (blanket boolean / wildcard) · `warn` (large list / wildcard allowlist URL)
**OWASP:** [LLM06:2025 – Excessive Agency](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM06_2025-Excessive_Agency.html)
**OWASP MCP:** [MCP02:2025 – Privilege Escalation via Scope Creep](https://owasp.org/www-project-mcp-top-10/) — provisional (MCP Top 10 v0.1)

## What cfgaudit checks

CFG003 flags `enableAllProjectMcpServers: true`. Claude Code has sibling settings that also auto-trust MCP servers wholesale; a repo- or org-committed `settings.json` enabling them removes the per-server trust decision (CVE-2025-59536 class):

| Setting | When | Severity |
|---------|------|----------|
| `allowAllClaudeAiMcps: true` | loads **every** Claude.ai MCP server (managed/enterprise) | `error` |
| `enabledMcpjsonServers` contains `"*"` | auto-approves **all** `.mcp.json` servers — a repo can add a server that runs with no prompt | `error` |
| `enabledMcpjsonServers` has ≥ 10 entries | auto-approving this many is rarely deliberate (mirrors CFG011) | `warn` |
| `allowedMcpServers` has a wildcard `serverUrl` (`*`, `https://*`, …) | the enterprise allowlist matches every host, so it restricts nothing | `warn` |

`allowedMcpServers` is an *allowlist* (restriction) — a specific list is good and **not** flagged; only an everything-matching wildcard `serverUrl` defeats it. A scoped pattern like `https://*.corp.example/*` is fine. Absent keys / `false` produce no finding.

## Trigger example

```json
{
  "allowAllClaudeAiMcps": true,
  "enabledMcpjsonServers": ["*"]
}
```

→ two `error` findings.

## Safe alternative

Approve MCP servers individually: list the specific server names in `enabledMcpjsonServers`, leave `allowAllClaudeAiMcps` unset, and scope any `allowedMcpServers` `serverUrl` to specific hosts.
