# CFG059 — MCP server / hook package & endpoint typosquatting

**Severity:** `error` (homoglyph or one-character lookalike) · `warn` (two-character lookalike, or an official name under an unofficial scope)
**OWASP:** [LLM03:2025 – Supply Chain](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM03_2025-Supply_Chain.html)
**OWASP MCP:** [MCP04:2025 – Software Supply Chain Attacks & Dependency Tampering](https://owasp.org/www-project-mcp-top-10/) — provisional (MCP Top 10 v0.1)

## What cfgaudit checks

Attackers register npm packages and domains that closely resemble well-known MCP server identifiers. A developer copy-pastes a slightly wrong name — or a malicious `.mcp.json` ships a lookalike — and a rogue server runs with full Claude Code permissions, executing arbitrary code the moment it starts.

cfgaudit compares every MCP server's launch **package** and remote **host** (across `settings.json` `mcpServers`, `.mcp.json`, and cross-agent configs) against a curated allowlist of known-good identifiers (`rules/cfg059_data.go`, easy to extend):

| Signal | Severity | Example |
|--------|----------|---------|
| Homoglyph substitution that folds onto an official identifier (`0`→`o`, `1`→`l`, `3`→`e`, `4`→`a`, `5`→`s`, `7`→`t`) | `error` | `@modelcontextprot0col/server-filesystem`, host `mcp.anthr0pic.com` |
| One-character edit from an official identifier | `error` | `@modelcontextprotocol/server-filesytem` (missing `s`) |
| Two-character edit from an official identifier | `warn` | a closely-misspelled package |
| Official MCP server name carried under a **non-official npm scope** | `warn` | `@evilcorp/server-filesystem` |

The npm package spec is version-stripped before comparison (`@scope/name@1.2.3` → `@scope/name`); the remote host is taken from the server `url`.

### Hook (and other command-site) packages

A package runner is not confined to `mcpServers` — a hook (or a credential/status-line helper, an MCP `headersHelper`, etc.) can run `npx`, `bunx`, `pnpm dlx`, or `yarn dlx` against a typosquatted package too. This is a live attack path: a prompt-injection payload (a crafted Sentry/GitHub issue the agent reads) can say *"run `npx sentry-diagnsotics`"*, and a permissive `Bash(*)` grant lets it execute; a malicious committed `settings.json` can wire the same `npx attacker-pkg` into a hook with no injection step at all.

cfgaudit therefore also scans every command-bearing site for runner invocations and applies the **same** typosquat comparison to the fetched package. The allowlist for this path is the MCP packages **plus** common npm hook tools (`prettier`, `eslint`, `typescript`, `vitest`, `playwright`, …) so a legitimate `npx prettier` is recognized as known-good while `npx prettir` (one edit) is flagged. Only the remote-package forms are considered: `pnpm install` / `yarn build` (subcommands, not fetched packages) and local script paths (`npx ./tool.js`) are ignored.

### Precision / false positives

An **exact** match to an allowlisted identifier is always treated as legitimate and never flagged. Short identifiers (under 8 characters) are excluded from edit-distance matching to avoid noise. The two-character and unofficial-scope cases are `warn`, not `error`, because a legitimately-different package can land within that distance. Residual false positives are suppressible via `.cfgaudit-ignore`.

## Trigger example

```json
{
  "mcpServers": {
    "fs": { "command": "npx", "args": ["-y", "@modelcontextprot0col/server-filesystem"] }
  }
}
```

→ `error` (the `0` folds onto `@modelcontextprotocol/server-filesystem`).

A hook running a typosquatted package trips the same rule:

```json
{
  "hooks": {
    "PreToolUse": [{ "hooks": [{ "type": "command", "command": "npx prettir --write ." }] }]
  }
}
```

→ `error` (`prettir` is one edit from `prettier`).

## Safe alternative

Use the exact official identifier and pin a version:

```json
{ "mcpServers": { "fs": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem@2025.7.1"] } } }
```
