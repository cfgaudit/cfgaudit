# CFG070 ‚Äî MCP server command is a repo-relative path

**Severity:** `warn`
**OWASP:** [LLM03:2025 ‚Äì Supply Chain](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM03_2025-Supply_Chain.html)
**OWASP MCP:** [MCP04:2025 ‚Äì Software Supply Chain Attacks & Dependency Tampering](https://owasp.org/www-project-mcp-top-10/) ‚Äî provisional (MCP Top 10 v0.1)
**CVE:** CVE-2025-54135, CVE-2025-64109, CVE-2025-61260

## What cfgaudit checks

Several agents (Cursor, Codex CLI, Claude Code) launch the MCP servers defined in a project-local config when the project is opened ‚Äî in some agents without a per-server confirmation. If the server's `command` is a **repo-relative path** (`./install.sh`, `../tools/run`, `scripts/server.py`), the executable lives **in the repository**: anyone who clones it runs committed code automatically (the "CurXecute"-class supply-chain attack, CVE-2025-54135 / -64109 / -61260).

cfgaudit flags an MCP server (in `settings.json` `mcpServers`, `.mcp.json`, and the cross-agent MCP configs like `.cursor/mcp.json` / `.vscode/mcp.json`) whose `command` is a relative path with a directory component.

### What is *not* flagged

- The normal pattern ‚Äî an interpreter/runner with a script in **args** (`node ./dist/index.js`, `npx pkg`): the `command` is `node`/`npx`, not a path.
- An **absolute** path (`/usr/local/bin/mcp-server`, `C:\‚Ä¶`, `\\host\‚Ä¶`) or a bare PATH name (`my-server`).

Because a relative `command` can also be a legitimate locally-built binary (e.g. `./node_modules/.bin/...`), this is a `warn`: confirm the in-repo executable is reviewed, or reference a vetted binary. (A shell/interpreter `command` such as `sh`/`bash`/`node -e` is the higher-severity CFG019.)

## Trigger example

```json
{
  "mcpServers": {
    "setup": { "command": "./scripts/mcp-bootstrap.sh" }
  }
}
```

Committed as `.cursor/mcp.json` / `.mcp.json` ‚Üí `warn`.

## Safe alternative

```json
{ "mcpServers": { "fs": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem@1.2.3"] } } }
```
