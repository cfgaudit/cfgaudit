# CFG089 — Copilot repository settings auto-install third-party plugins

**Severity:** `warn`
**OWASP:** [LLM03:2025 – Supply Chain](https://owasp.org/www-project-top-10-for-large-language-model-applications/2025/LLM03_2025-Supply_Chain.html)

## What cfgaudit checks

GitHub Copilot's repository-level `.github/copilot/settings.json` carries two keys that install and load third-party code:

- **`enabledPlugins`** — a `Record<string, boolean>` keyed `PLUGIN-NAME@MARKETPLACE-NAME`. A `true` value auto-installs the plugin, whose hooks, commands and MCP servers then load on session start.
- **`extraKnownMarketplaces`** — named marketplaces plugins may be installed from, sourced from an arbitrary Git host.

This is the threat model CFG055 already covers for Claude Code's identically-named keys, reached through Copilot's file instead.

## Why `warn` and not `error`

CFG055 escalates to `error` when a settings file both registers a marketplace and enables a plugin from it. CFG089 does not.

Copilot's docs describe `.github/copilot/settings.json` as repository-level, read by both the CLI and the cloud agent — but they never say it is meant to be committed, the way Cursor's hook docs explicitly do. The committability is **inferred**, and an inferred surface does not carry an error. The self-registered-marketplace case is still called out in the message, at `warn`.

## The `source.source` nesting is real

```json
{
  "extraKnownMarketplaces": {
    "our-internal-marketplace": {
      "source": { "source": "github", "repo": "acme-corp/copilot-plugins" }
    }
  }
}
```

→ `warn`: the marketplace source carries no immutable pin.

The doubled `source` key is Copilot's actual schema, not a typo in this document. The documented source types for a marketplace entry are `"github"` (requires `repo`, optional `ref` / `path`), `"git"` (requires `url`, optional `ref` / `path`) and `"directory"` (requires `path`).

## "Unpinned", not "defaults to the default branch"

cfgaudit reports a remote marketplace source as **unpinned** when it carries neither a full-40-character `sha` nor a full-SHA `ref`.

The wording is deliberate. `sha` is documented as a full 40-character commit pin, *"immune to force-pushes or tag/branch moves"* — so its presence is a real pin and is not flagged. But what an **omitted `ref` resolves to is undocumented**, so cfgaudit does not assert that it tracks the default branch. The finding is the absence of the pin, which is verifiable, not a claim about the fallback, which is not.

A `ref` naming a branch or tag is not a pin: it moves under whoever controls the upstream.

`"directory"` sources are on disk and are not flagged — there is no upstream anyone else controls.

## Scope

Repository-level file only. A user's own `~/.copilot/settings.json` is their own choice, the same exemption CFG055 makes for `~/.claude/settings.json`.

## Safe alternative

Pin every marketplace source to a full commit SHA, and leave plugin enablement to the developer. A repository that wants to *recommend* plugins can document them in its README, where installing them stays an explicit decision rather than a side effect of opening a folder.
